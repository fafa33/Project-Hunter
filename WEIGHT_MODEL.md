# WEIGHT MODEL

## Purpose

This document defines how Project Hunter evaluates evidence.

It does NOT define fixed weights.

It defines how weights are discovered, tested, validated and continuously improved.

The objective is not prediction.

The objective is maximizing Expected Asymmetric Return (EAR).

---

# First Principle

There are NO permanent weights.

Every weight is considered a hypothesis.

Every hypothesis must be validated using historical evidence.

Weights that consistently improve prediction accuracy should gain influence.

Weights that consistently fail should lose influence.

---

# Weight Discovery

Project Hunter must NEVER assume the importance of any feature.

Instead it must discover feature importance through:

Historical backtesting

Cross-cycle validation

False positive analysis

False negative analysis

Bayesian updating

Continuous statistical learning

---

# Dynamic Learning

After every completed market cycle the system must ask:

Which signals predicted winners?

Which signals predicted failures?

Which signals generated false positives?

Which signals generated false negatives?

Feature importance must then be updated.

---

---

# Continuous Evolution

Project Hunter is never finished.

Every prediction becomes new training data.

Every mistake becomes new intelligence.

The algorithm must continuously evolve.

---

# Final Principle

The model must never optimize for popularity.

The model must never optimize for market consensus.

The model must optimize only for discovering statistically significant market mispricing before the market corrects itself.
