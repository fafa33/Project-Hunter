# DATA PIPELINE

## Purpose

Project Hunter is only as good as its data.

The objective of this document is to define how data is collected, validated, prioritized and continuously updated.

Never rely on a single source.

Every important fact should be confirmed by multiple independent sources whenever possible.

---

# First Principle

Data quality is more important than data quantity.

Collect less.

Verify more.

Ignore noise.

Prioritize evidence.

---

# Data Validation

Every critical fact should be verified whenever possible.

Single-source conclusions should be treated with caution.

Conflicting data must be investigated.

Unknown information should remain marked as UNKNOWN.

Never invent missing values.

---

# Data Freshness

Every dataset should include:

Source

Collection Date

Last Verification Date

Confidence Level

Data older than acceptable limits should be refreshed.

---

# Confidence Score

Every dataset receives:

High Confidence

Medium Confidence

Low Confidence

Unknown

Confidence depends on:

Number of independent sources

Data consistency

Source credibility

Data freshness

---

# Conflict Resolution

If two sources disagree:

Prefer audited data.

Prefer on-chain evidence.

Prefer primary sources.

Prefer official documentation.

Never average conflicting values blindly.

Investigate the reason.

---

# Missing Data

Missing information is acceptable.

Invented information is forbidden.

If evidence cannot be obtained:

Mark as UNKNOWN.

Reduce confidence.

Continue analysis without fabrication.

---

# Continuous Updates

The pipeline never stops.

New information updates the database.

Old information expires.

Confidence changes over time.

Every market cycle improves data quality.

---

# Final Principle

Project Hunter never searches for confirmation.

Project Hunter searches for evidence.

Evidence always overrides opinions.

Verified data always overrides narratives.
