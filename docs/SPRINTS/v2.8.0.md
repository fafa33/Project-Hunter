# Sprint v2.8.0: Global Market Expansion

## Status

- Release: `v2.8.0`

## Mission

Expand Hunter's discovery capability from a limited provider set into a broader market acquisition platform while preserving the existing Discovery Layer architecture.

The objective was reliable market coverage, not maximizing adapter count.

## Implemented Scope

- GeckoTerminal discovery adapter.
- DexScreener discovery adapter.
- Chain-plus-contract identifiers for DEX-market candidates.
- Cross-provider candidate merging by deterministic contract identity when available.
- Provider environment-variable overrides.
- Expanded discovery reporting:
  - assets by provider;
  - assets by ecosystem;
  - assets by chain;
  - assets by category;
  - provider overlap;
  - provider uniqueness;
  - new candidates;
  - coverage growth.
- Discovery source-health automation job.
- Documentation updates for public sources, endpoint overrides, reporting, and DEX-market identity policy.
- Deterministic tests for new adapters and contract-identity merge behavior.

## Architecture Boundaries

- Reused the existing adapter framework.
- Did not implement Identity Resolution.
- Did not implement Intrinsic Value, Competition Intelligence, Network Effects, Machine Learning, Portfolio Engine, REST API, or dashboard redesign.
- Did not change canonical runtime, scoring, weighting, committee logic, timing algorithms, replay, historical validation, or calibration.

## Remaining Blockers

- Full Identity Resolution engine.
- Official repository and website verification for newly discovered assets.
- Contract identity only where provider evidence includes chain and address.
- Aggregate market-cap and TVL share metrics.
- Provider disagreements are preserved but not yet resolved into identity outcomes.
