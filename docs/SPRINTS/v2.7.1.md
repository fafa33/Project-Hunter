## Mission

Harden the Discovery Layer created in `v2.7.0` so it behaves deterministically under live provider data, provider failure, repeated sync, and larger imports.

## Implemented Scope

- Deterministic batch registry upserts.
- One SQLite transaction per provider import.
- Canonical seeded identity preservation when provider identifiers merge into seeded candidates.
- Provider retry and backoff.
- Explicit provider failure isolation.
- Deterministic screening outcomes: `advanced`, `deferred`, and `rejected`.
- Queue-score clamping.
- Expanded discovery coverage reporting.
- Additional deterministic registry tests.

## Architecture Boundaries

- Did not redesign the Discovery Layer.
- Did not implement Identity Resolution.
- Did not change canonical scoring, committee, timing, replay, historical validation, or calibration behavior.

## Weaknesses Fixed

- Provider-specific slugs no longer overwrite canonical seeded slugs.
- Provider metadata no longer overwrites seeded sector/type when canonical seeded values already exist.
- Thin-evidence candidates no longer automatically advance.
- Blocked, scam, spam, and impersonation markers reject deterministically.
- Provider outages no longer create false zero-activity or corrupted registry states.
- Large candidate imports no longer require one connection per candidate.
