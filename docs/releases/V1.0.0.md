## Project Overview

Project Hunter V1 is a deterministic, explainable crypto research platform for discovering evidence-backed asymmetric opportunities before they become obvious to the market.

V1 is analytical infrastructure only. It does not execute trades, allocate portfolios, publish price targets, or call external AI systems.

## Architecture Overview

V1 is built around strict separation of responsibilities:

- Pipeline orchestration controls execution order.
- Intelligence engines produce standardized evidence-backed intelligence.
- Fusion combines persisted intelligence into explainable fused outputs.
- Opportunity Timing evaluates evidence-backed timing windows from persisted Fusion outputs.
- Probability, Pattern Matching, and Technology Necessity consume persisted downstream records.
- Investment Committee consolidates persisted outputs into final explainable committee decisions.
- Automation schedules execution without analytical logic.
- Dashboard Foundation displays persisted outputs only.
- SQLAlchemy remains isolated inside the persistence layer.

## Complete Analytical Pipeline

Discovery -> Evidence -> Validation -> Scoring -> Valuation -> Mispricing -> Asymmetry -> Whales -> Macro -> Future Demand -> Opportunity Timing -> Probability -> Pattern Matching -> Technology Necessity -> Investment Committee -> Reports -> Ranking -> Dashboard.

## Persistence Architecture

V1 uses immutable analytical record contracts, repository interfaces, UnitOfWork boundaries, SQL repositories, and deterministic serialization. Operational attempts are separate from analytical run identity. Repeated persistence of identical analytical records is idempotent.

## Automation Architecture

Automation schedules and invokes pipeline execution. It does not contain scoring, ranking, committee, timing, or intelligence logic. Replay jobs require explicit timezone-aware `as_of` semantics.

## Dashboard Foundation

Dashboard Foundation is a presentation layer over persisted records. It does not compute analytical decisions.

## Investment Committee

The Investment Committee Engine consumes persisted upstream outputs and produces deterministic committee assessments, votes, champion snapshots, explainability, invalidation conditions, and ranking-compatible outputs.

## Pipeline Architecture

PipelineOrchestrator controls how analysis runs. Plugins and intelligence engines remain replaceable. Persistence integration is optional and backward compatible.
