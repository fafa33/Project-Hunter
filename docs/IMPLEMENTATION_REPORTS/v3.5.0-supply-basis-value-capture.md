## Authority Flow

```text
Registered Evidence Source
→ SupplyAndValueCaptureService
→ SupplyAndValueCaptureRepository
→ Dedicated SQL-authoritative persistence
```

The service owns registry authorization, canonical identity enforcement, evidence linkage, deterministic identity and content hashing, correction authorization, strict-known replay, and write-plan authorization.

The repository owns atomic immutable persistence and exact retrieval only.

## Supply Basis

The implementation preserves distinct supply categories including circulating, total, max, fully diluted, issued, treasury-held, locked, vested-but-unreleased, burned, and protocol-owned supply.

No category automatically satisfies another category. No cross-representation aggregation is implemented.

## Value Capture

The implementation preserves explicit rule types including burn, buyback, buyback-and-burn, staking distribution, fee distribution, revenue distribution, protocol-owned-liquidity accrual, redemption entitlement, collateral entitlement, governance-directed distribution, no direct value capture, unavailable, and unsupported.

A rule cannot persist as authoritative without registry capability and matching accepted evidence.

## Replay

Strict-known selection requires:

- exact entity, economic claim, representation, and category scope;
- `effective_at <= effective_as_of`;
- `recorded_at <= known_by`;
- `known_at <= known_by`;
- accepted quality;
- no open or contested conflict.

There is no latest fallback, current-state fallback, manual fallback, post-cutoff substitution, or cross-representation substitution.

## Focused Tests

Focused tests cover:

- unregistered and disabled source rejection;

- unauthorized repository mutation rejection;
- evidence, supply, and rule idempotency;
- evidence existence and identity matching;
- strict-known cutoffs;

## Explicit Exclusions Preserved

This implementation does not activate or modify:

- fair value;
- intrinsic value;
- valuation methodology;
- comparative valuation;
- peer selection;
- mispricing;
- asymmetry;
- Opportunity scoring or ranking;
- Portfolio Intelligence;
- Dashboard-side calculations;
- Market Validation formulas, weights, Timing, committee, reports, or scalar inputs.

The Market Validation inputs `valuation`, `comparative_valuation`, `mispricing`, and `asymmetry` remain unavailable.
