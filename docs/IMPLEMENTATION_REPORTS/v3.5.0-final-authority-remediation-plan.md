# v3.5.0 Final Authority Remediation Plan

This document records the implementation plan for the three blockers identified by the post-merge audit. Production changes are implemented on a separate remediation branch.

Required controls:

- persistent provider signing identity supplied by runtime configuration rather than generated per process;
- persisted receipt signature and signing-key identity;
- repository-side cryptographic verification before every authoritative write;
- no API that returns an unguarded writer closure;
- deterministic restart, tampering, forged-provider, and direct-write tests.
