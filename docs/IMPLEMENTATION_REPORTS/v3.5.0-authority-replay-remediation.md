# v3.5.0 Authority and Replay Remediation

## Scope

This remediation closes the blockers recorded in `docs/ARCHITECTURE_AUDITS/v3.5.0-supply-basis-value-capture.md` without activating valuation, mispricing, asymmetry, Opportunity, Portfolio, Dashboard calculations, or Market Validation scalar inputs.

## Implemented controls

- Added sealed, typed `ValueCaptureAcquisitionResult` records created only by `RegisteredValueCaptureProvider`.
- Service constructs canonical evidence, supply, and value-capture records from registry-authorized provider outcomes.
- Added acquisition IDs and raw payload hashes to persisted provenance.
- Enforced exact registry source, endpoint, parser, capability, identity, and representation checks.
- Enforced `effective_at <= recorded_at <= known_at` for all record families.
- Enforced evidence causal bounds before derived snapshot persistence.
- Enforced evidence/source/parser identity equivalence for derived records.
- Removed the repository public write-plan API and public authority attribute.
- Added service-token-gated repository commits.
- Enforced single-successor correction lineage, predecessor identity, correction chronology, and correction reason pairing.
- Updated strict-known replay to select the unsuperseded record as known at the replay cutoff.

## Deterministic tests

Tests cover:

- forged provider result rejection;
- absence of public repository mutation APIs;
- provider-bound idempotent persistence;
- future-known evidence rejection;
- source/parser mismatch rejection;
- temporal chronology rejection;
- branching correction rejection;
- replay selection before and after a correction becomes known.

## Verification requirement

The remediation is not complete until all repository quality gates pass and an independent audit confirms closure.
