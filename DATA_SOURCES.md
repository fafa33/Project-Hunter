# DATA SOURCES

## Objective

The Hunter Engine must never rely on a single source.

Every important claim must be validated using multiple independent sources whenever possible.

Sources must be ranked by reliability.

No marketing article should ever be treated as evidence.

# Reliability Rules

Each data point must receive one of the following confidence labels:

Verified

Highly Reliable

Reliable

Uncertain

Weak Evidence

Unverified

Speculative

Unknown

---

# Conflict Resolution

If two sources disagree:

Prefer on-chain data.

Prefer official documentation.

Prefer audited financial data.

Never average conflicting information.

Always explain the disagreement.

---

# Missing Data

If important data is unavailable:

State that it is unknown.

Never invent values.

Never estimate without evidence.

Missing data is preferable to false certainty.

---

# Continuous Updating

Every source must be refreshed periodically.

Historical values must be preserved.

Changes over time are often more important than current values.
